# 起動トリガー

grow-wiki スキルを起動するユーザー発話のパターン。

## 明示的な指示（即座に ingest へ）

以下のフレーズは ingest フローを直接起動する。ただし書き込み前に必ずプレビューと承認を挟む。

- 「grow-wiki に保存」「grow-wiki に追加」
- 「wiki に保存」「wiki に追加」「wiki 化」
- 「この URL を取り込んで」「この URL を wiki に」
- 「〇〇ページを更新して」「〇〇を wiki に反映」
- 「grow-wiki で lint」「grow-wiki で health check」「grow-wiki の index を再生成」

## キーワード自動提案（ユーザーに提案 → 承認後に起動）

以下のフレーズが会話に現れた場合、「grow-wiki に保存しましょうか？」と提案する。ユーザーが yes と答えたら ingest 開始。no と言われたら提案を繰り返さない。

- 「メモっておいて」「メモしとく」
- 「覚えておいて」「記憶しておいて」
- 「あとで読む」「後で読もう」
- 「ブックマーク」「ブクマ」
- 「保存しとく」「残しておく」
- 「知識として残す」「ナレッジに」

### 提案プロンプトのテンプレ

```
「grow-wiki に保存しましょうか？ 対象：
- タイトル案: <タイトル>
- type: <source-conversation / source-url / entity / concept>
- 格納先: <path>

保存して良ければ「はい」「ok」などとお返事ください。」
```

## 過剰発火の回避

- URL が単に会話文中に引用されただけでは自動提案しない（明示指示時のみ）
- 「覚えておいて」が技術的文脈でなく雑談文脈（「名前覚えておいてね」など）の場合は提案しない
- 同じ会話内で一度「no」と答えたキーワードに対しては、そのセッション内で再提案しない
- ingest 中・承認待ち中に新たなトリガーが発火した場合は、現フローを完了してから判定

## 判定順（書き込み系）

1. 明示指示パターンのマッチを最優先（即 ingest へ）
2. 明示でなければキーワード自動提案のマッチを評価
3. どちらもヒットしなければ通常会話として処理

---

## 参照トリガー（Q&A 用）

ユーザーから質問・相談があった時、grow-wiki に蓄積された情報を回答に活用するためのトリガー。詳細フローは [query-flow.md](query-flow.md)。

### 明示的な指示（即 query へ）

以下は query フローを即起動。

- 「grow-wiki に聞いて」「grow-wiki を参照して」「grow-wiki から探して」
- 「wiki を見て」「wiki を見て答えて」「wiki から教えて」
- 「蓄積した知識から」「ナレッジから」「過去のメモから」
- 「以前の話では？」「前に調べた〇〇」「前に書いた〇〇」「前のメモは？」
- 「grow-wiki に〇〇についてある？」「〇〇について wiki では？」

### キーワード自動提案

以下の条件を**すべて**満たす時、「grow-wiki の [[〇〇]] に関連情報があります。参照しますか？」と提案する。

- ユーザーの発話中に**名詞句**が含まれる
- その名詞句が vault の page title / aliases / synonyms のいずれかと**完全一致**する、または tags のうち 2 つ以上と重なる
- マッチする語が **3 文字以上**（短い一般語での誤発火を避ける）
- 同一会話で同じ提案に既に「no」と答えていない
- 既にその話題で参照済みではない

短い一般語（「設計」「方法」「技術」「パターン」等）は vault の tag にあっても自動提案しない。明示指示で呼ぶ。

### 判定順（参照系）

1. 書き込み系トリガーが同時発火していたら書き込みを優先（ingest 後に query でも可）
2. 参照系の明示指示があれば即 query フローへ
3. 明示指示でなくてもキーワード自動提案条件を満たせば提案
4. どれも発火しなければ通常回答（vault は参照しない）

### 過剰発火の回避

- 同一会話内で一度「no」と答えた話題は再提案しない
- 通常 Q&A に毎回割り込まない（1 会話 2-3 回までを目安に、発火を自制）
- マッチ数が多すぎる時（5 件超）は具体語が不足しているので提案を見送り「どの文脈か」を聞き返す
