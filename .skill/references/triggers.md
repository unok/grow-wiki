# 起動トリガー

grow-wiki スキルを起動するユーザー発話のパターン。

## 明示的な指示（即座に ingest へ）

以下のフレーズは ingest フローを直接起動する。ただし書き込み前に必ずプレビューと承認を挟む。

- 「grow-wiki に保存」「grow-wiki に追加」
- 「wiki に保存」「wiki に追加」「wiki 化」
- 「この URL を取り込んで」「この URL を wiki に」
- 「〇〇ページを更新して」「〇〇を wiki に反映」
- 「grow-wiki で lint」「grow-wiki で health check」「grow-wiki の index を再生成」

## キーワード自動提案（ユーザーに提案 → 承認後に起動）

以下のフレーズが会話に現れた場合、「grow-wiki に保存しましょうか？」と提案する。ユーザーが yes と答えたら ingest 開始。no と言われたら提案を繰り返さない。

- 「メモっておいて」「メモしとく」
- 「覚えておいて」「記憶しておいて」
- 「あとで読む」「後で読もう」
- 「ブックマーク」「ブクマ」
- 「保存しとく」「残しておく」
- 「知識として残す」「ナレッジに」

### 提案プロンプトのテンプレ

```
「grow-wiki に保存しましょうか？ 対象：
- タイトル案: <タイトル>
- type: <source-conversation / source-url / entity / concept>
- 格納先: <path>

保存して良ければ「はい」「ok」などとお返事ください。」
```

## 過剰発火の回避

- URL が単に会話文中に引用されただけでは自動提案しない（明示指示時のみ）
- 「覚えておいて」が技術的文脈でなく雑談文脈（「名前覚えておいてね」など）の場合は提案しない
- 同じ会話内で一度「no」と答えたキーワードに対しては、そのセッション内で再提案しない
- ingest 中・承認待ち中に新たなトリガーが発火した場合は、現フローを完了してから判定

## 判定順

1. 明示指示パターンのマッチを最優先（即 ingest へ）
2. 明示でなければキーワード自動提案のマッチを評価
3. どちらもヒットしなければ通常会話として処理
