# 命名規則

ファイル名・フォルダ名・タイトルの付け方。

## ファイル名

### 基本方針

- **日本語 OK**。Obsidian は日本語ファイル名を問題なく扱う
- ファイル名はそのまま `[[wikilink]]` 解決に使われるので、見出しとして自然な形に
- `.md` 拡張子を必ず付ける

### 禁止文字

以下はファイル名に含めない（ファイルシステム/Obsidian 両対応のため）:

```
/  \  :  *  ?  "  <  >  |
```

加えて:

- 先頭・末尾の空白
- 先頭のドット `.`（隠しファイル化される）
- 制御文字 `\x00-\x1f`

### 変換ルール

- 全角スペース `　` → ハイフン `-`
- 半角スペース `  `（連続） → 単一の半角スペース
- 禁止文字はハイフン `-` に置換または削除
- 長さ 100 文字を超える場合は省略（意味が残るように後半を `…` に）

### 衝突対処

同一ファイル名が既に存在する場合:

1. **文脈サフィックス（推奨）**: `Claude (Anthropic).md` vs `Claude (人名).md`
2. 同義の別名が明確にある場合はそちらを採用: `React (React Native).md` → `React Native.md`
3. 最終手段: `-2`, `-3` などの数字サフィックス（意味がないため可能な限り避ける）

## タイトル（frontmatter の `title`）

- ファイル名と一致させるのが基本
- 例外: ファイル名に禁止文字変換が入った場合、元の自然な表記を `title` に残す
- 例: ファイル名 `React-Native.md` / title `React Native`

## フォルダ名

- 日本語 OK だが、英語・カタカナ推奨（URL エンコード/CLI 扱いが楽）
- `misc/` は再編成時の余剰置き場として予約
- rebalance で生成されるサブフォルダ名は代表タグをそのまま使う

## 英略語・製品名

- そのまま記す: `React` / `Obsidian` / `Claude` / `GitHub`
- 日本語化しない（統一感と検索性のため）

## 日付ベースのファイル名は使わない

- daily notes 風の `2026-04-20.md` は使わない（sources/conversations/ は日付ベース命名ではない）
- 会話 ingest のファイル名は**内容に基づくタイトル**にする（`created` フィールドで日付は分かる）

## 例

| 内容 | OK | NG |
|---|---|---|
| React Hooks の解説 | `React Hooks.md` | `react-hooks.md`（英語圏でも OK だが日本語 vault では title 一致優先） |
| useMemo と useCallback の違い | `useMemo と useCallback の使い分け.md` | `useMemo/useCallback.md`（禁止文字 `/`） |
| 2026-04-20 の雑談 | 内容に応じた名前に | `2026-04-20 雑談.md`（内容不明瞭） |
| Claude 社 vs Claude 人名 | `Claude (Anthropic).md` / `Claude (人名).md` | `Claude.md`（曖昧） |
