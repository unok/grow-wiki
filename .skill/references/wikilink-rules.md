# wikilink 付与ルール

## 原則

- **`[[ファイル名]]` 形式のみ使用**。`[text](path.md)` や絶対パスは使わない
- Obsidian のリンク解決は**ファイル名ベース**なので、階層移動に自動追従する
- 自己参照（自分自身のファイル名への `[[]]`）は禁止

## 付与対象

会話/URL 本文をスキャンし、以下に該当する語を `[[...]]` で囲む:

1. 既存ページの `title` と完全一致する語
2. 既存ページの `aliases` に含まれる語
3. 既存ページの `synonyms` に含まれる語
4. これから同 ingest で新規作成する予定の entity/concept のタイトル

## 解決ロジック

1. `list-pages.sh` の JSON から `{ file_basename, title, aliases, synonyms }` のマップを作る
2. 本文を単語/フレーズ単位でスキャン（日本語は形態素解析せず、完全一致と最長一致で OK）
3. マッチしたら `[[元の語]]` に置換。ただしコードブロック (```...```) 内、インラインコード `` `...` `` 内、既存の `[[...]]` 内は除外
4. 曖昧な場合（同一語が複数 entity に一致する等）は付与せずユーザーに確認

## 表示テキストのエイリアス記法

Obsidian は `[[ファイル名|表示テキスト]]` をサポートする。本文中で異なる表記を使いたい場合に使う:

```
[[Claude (Anthropic)|Claude]]
```

ただし多用は避ける（可読性が下がる）。

## 新規エントリを作るタイミング

会話/URL で頻出する語が既存ページになく、かつ以下のいずれかに当たる場合は entity/concept を新規作成してリンクする:

- 固有名詞（人物・製品・ツール・組織） → `entity`
- 専門用語・概念 → `concept`

一度の ingest で大量（> 10）の新規ページを作らない。多すぎる場合はユーザーに優先順位を確認。

## 避けるケース

- 一般的な名詞（「技術」「方法」「理由」など）はリンクしない
- 動詞・形容詞は原則リンクしない（名詞形がページとしてあれば別）
- 否定文脈・仮定文脈で言及しただけの場合はリンクしない

## リンク切れ対策

- ページ移動時は Obsidian が自動追従するが、Claude が直接ファイルを `mv` した場合は追従しない
- スキルからの移動は必ず `rebuild-index.sh` の後に `health-check.sh` を実行して broken link = 0 を確認
- broken link が出た場合は `log.md` に `lint-error` を記録しユーザーに通知
